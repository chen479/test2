# 贡献指南

感谢您对本项目的关注和支持！我们欢迎各种形式的贡献，包括但不限于代码、文档、示例和建议。

## 贡献流程

1. **Fork 仓库**：在GitHub上fork本仓库到您自己的账号
2. **克隆仓库**：将fork后的仓库克隆到本地
3. **创建分支**：从 `main` 分支创建一个新的功能分支
4. **进行修改**：在新分支上进行您的修改
5. **提交代码**：使用规范的提交信息提交您的更改
6. **推送到远程**：将您的分支推送到GitHub
7. **创建PR**：在GitHub上创建一个Pull Request，描述您的更改

## 分支管理

- **main**：主分支，包含稳定的代码和文档
- **feature/**：功能分支，用于开发新功能
- **bugfix/**：修复分支，用于修复bug
- **docs/**：文档分支，用于更新文档

## 提交规范

提交信息应遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范，格式如下：

```
<类型>(可选的范围): <描述>

[可选的正文]

[可选的页脚]
```

### 类型

- **feat**：新功能
- **fix**：修复bug
- **docs**：文档更新
- **style**：代码风格调整
- **refactor**：代码重构
- **test**：测试相关
- **chore**：构建过程或辅助工具的变动

### 示例

```
feat(codes): 添加FFT示例代码

添加了基于Python和Matlab的FFT实现示例，包括快速傅里叶变换的基本原理和应用

Closes #123
```

## 代码规范

### Python代码
- 遵循 [PEP 8](https://www.python.org/dev/peps/pep-0008/) 代码风格
- 使用4个空格进行缩进
- 每行不超过80个字符
- 函数和方法应包含文档字符串

### Matlab代码
- 使用有意义的变量名和函数名
- 添加适当的注释
- 遵循Matlab的代码风格指南

### 文档
- 使用Markdown格式编写文档
- 保持文档结构清晰
- 代码示例应包含注释和说明

## 测试

- 为新功能添加测试用例
- 确保所有测试通过
- 测试代码应放在适当的位置

## 问题反馈

如果您发现bug或有新功能建议，请在GitHub上创建Issue，描述问题的详细信息。

## 行为准则

请尊重其他贡献者，保持友好和专业的沟通。

## 许可证

通过贡献代码，您同意您的贡献将在 [MIT License](LICENSE) 下发布。
